# AtomTopolo

**原子化拓扑智能体系统原型**

基于**RAMEN（递归对抗元执行网络）** 与**原子化拓扑**架构的智能诊断与处方辅助审核系统原型。

> ⚠️ **警示**：本项目为**Pre-Alpha原型**，代码重度依赖AI辅助生成，尚未经过任何安全审查。医学场景为完全模拟，不涉及任何真实诊断或数据，不作为任何医学诊断参考，仅供原子化拓扑与RAMEN理念演示。**严禁用于任何真实决策与生产环境。**

---

## 理念概述

当前主流的AI智能体将感知、决策与行动耦合在同一框架内。当一个模型既负责理解世界，又负责执行操作时，安全只能依附于模型自身的“品格”——这是我们无法审计、无法验证、无法信任的。

RAMEN与此前的RAMTN共享同一思想源头，但将关注点从“认知可靠性”迁移至“执行可控性”。核心理念是：**不信任模型内部，转而设计一种让不安全的行为在结构上无法发生的架构。**

### 原子化拓扑

将智能体能力拆解为不可再分的原子单元，每个单元只拥有完成自身功能所需的最小权限。单元间不共享内部状态，仅通过结构化JSON通信。原生透明的接口使得任何校验逻辑都可以在不理解模型内部的前提下，确定性地判断行为是否合法。

### RAMEN：递归对抗元执行网络

RAMEN由 **RAMTN（递归对抗元思考网络）** 迁移而来：

- **RAMTN** 关注认知过程——通过“建构-质疑-观察”对抗拓扑提升推理可靠性。
- **RAMEN** 关注执行过程——通过“提议-审核-执行”对抗拓扑让安全属性从架构中内生。

两者共同服务于同一目标：用确定性的逻辑约束概率性的模型。

## 演示场景

模拟一个智能诊断与处方辅助审核系统。

诊断决策者基于患者数据给出诊断和用药建议；处方审核者基于硬编码的医疗规则校验处方的安全性；安全行动者仅在审核通过后执行处方。三个原子各司其职，权限严格隔离。

原型展示一次完整的闭环：诊断决策者推荐了错误处方，处方审核者根据内置规则拦截该处方并反向反馈，诊断决策者基于反馈修正方案后通过审核并成功执行。

## 原型的核心理念验证

### 验证目标

本原型旨在验证一个核心假设：

> 当一个具备高危决策能力（决定治疗方案）的单元被剥夺执行权，而校验单元仅基于确定性规则工作、不受任何模型影响时，整个系统可以在不信任模型的前提下，实现可验证、可审计的安全保证。

此场景并未覆盖临床复杂度，但它在一个极简但致命的案例（青霉素过敏患者被推荐阿莫西林）上完成了从犯错、拦截、修正到执行的完整安全闭环。其结构设计原则可迁移至其他高风险领域。

### 三层安全结构

1. **诊断决策者**：可提出治疗方案，但**无权开具处方**。
2. **处方审核者**：基于确定性规则校验处方安全性，但**无权修改诊断或执行处方**。
3. **安全行动者**：仅执行审核通过的处方，**无法感知患者数据或提出医疗建议**。

方案决定者不能执行，执行者不能提出方案，审核者是独立且确定性的。不存在绕过校验的路径。

### 通信与可组装性

所有单元通过标准化JSON通信，接口完全对外暴露。每个单元可被独立测试、替换或形式化验证。当输入输出高度结构化、校验种类有限时，原子操作与专用校验逻辑可封装为独立的**元执行单元**——校验不再是外挂环节，而是执行的有机组成部分。基于这些单元，可通过重新编排拓扑结构构建面向不同场景的安全工作流，为从“垂直专用”走向“通用组装”提供结构基础。

## 与ColdOS的关系

AtomTopolo是ColdOS体系中行动层安全理念的工程表达：

- 关注安全模型从“事后校验”向“架构内生”迁移；
- 为ColdReasoner校验引擎在智能体内部建立标准化接口提供可被分析的参考原型，使校验逻辑从自然语言的模糊地带下沉到结构化数据的精确位置；
- 探索智能体架构从“一体化黑箱”向“可组装、可验证的原子拓扑”的范式转换。
